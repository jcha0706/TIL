1. 앞으로 git 프로젝트로 만들고 싶은 것 
배울 내용들 응용해, 다른 개발자들과 협업
클론해서 로컬 리포로 가져온 프로젝트 수정후 
2. 1주차 주요 개념 키워드 적어보기
-버전 관리: 새로 업데이트한 파일을 새로 생성해 각 버전 관리하지 않고,
	git에서 새로 업데이트된 부분을 자동 업데이트화 시킴
-git init (초기화): git에 새로운 원격 리포를 만들고 로컬 리포와 연결
-add/staging: 전체 프로젝트가 아닌 부분적인 또는, 몇개 파일의 변경을 선택해 저장
-commit: 소스트리에서 로컬에서 변경된 내용을 저장
-commit history: 업데이트 와 그외 해왔던 것
-repository: git으로 관리되는 프로젝트
-원격 리포: github 처럼 다른 곳에서 접속할 수 잇는 공간에 저장되어 있는 것
-로컬 리포: 내 컴퓨터에 저장되어 있는 리포지토리
-branch: 한 리포지토리에서 나눠놓은 여러 저장소 (원본 데이터 유지보수 가능)
-push: 변경 내용이 생긴후 커밋완료후 원격 리포에 변경사항을 업데이트 하는 행위
-pull: 원격 리포에서 로컬 리포로 내 파일들과 병합하는 행위
-clone: 총 프로젝트를 윈격 리포에서 가져오는 행위
-tracking: 로컬과 원격을 연결함
